Команди GIT

  • git config --global user.name blabla - встановлює ім'я, яке буде віджображатися в полі автора у виконуваних коммітів
  • git config --global user.email blabal@bla - встановлює адресу ел.пошти, яка буде віджображатися в полі автора у виконуваних коммітів
  • git init [назва проекту] - створює новий репозиторій із вказаним ім'ям
  • git status - вказує всі нові або змвнені файли, які потрібно зафіксувати
  • git add [файл] - індексує вказаний файл для майбутнього комміту
  • git commit -m "[повідомлення з описом]" - фіксує проіндексовані зміни і зберігає їх в історію версій
  • git log - історія коммітів для поточної гілки
  • git checkout [ім'я гілки] - переключається на вказану гілку і відновлює робочу директорію до її стану
  • git branch [ім'я гілки] - створює нову гілку
  • git push [віддалений репозиторій] [гілка] - загружає всі зміни локальної гілки в віддалений репозиторій
  • git pull - Загружає історію з віддаленого репозиторію і об'єднує її з локальною. pull = fetch + merge
  • git merge [віддалений репозиторій]/[гілка] - Вносить зміни з гілки віддаленого репозиторію в поточну гілку локального репозиторію
  • git clone [url-адрес] - клонує (зкачує) репозиторій разом з усією його історією змін
  • .gitignore - файл, в якому перераховані всі файли і директорії (папки), які будуть ігноруватися git
